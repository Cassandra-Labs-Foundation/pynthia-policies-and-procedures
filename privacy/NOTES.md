# Privacy — Drafting Notes

## Reference doc value
- `Acorns Example PP.md` — real-world GLBA "FACTS" notice (federal) plus heavy California state overlays. Especially useful because:
  - **GLBA FACTS template** at the top is mandatory for all financial institutions regardless of regulator → directly reusable.
  - **CCPA / CPRA** sections at the bottom apply under DFPI scope → good template.
  - **California Shine the Light** disclosure → good template.
  - Vermont / Nevada state variants → good model for our PR-12 (state variants) section.
- `Privacy Policy 6-2022.pdf` — additional template.

## DFPI scope
- GLBA-based privacy notice (federal) still required — no change.
- **CCPA / CPRA** applies — the Acorns doc has solid templates for the California Collection Notice, Shine the Light disclosure, and the rights/exercise sections. Use as model for PR-12 California variant.
- Nevada and Vermont state variants in Acorns doc serve as good models if Pynthia operates in those states.

## Canonical structure
Existing sections PR-01..PR-17 already cover:
- Notice lifecycle, opt-out, permissible disclosures, access/auth, accuracy, employee minimization, third-party oversight, secure disposal, incident response, recordkeeping, web posting, state variants (CCPA/VT/NV), anonymization, cookies, third-party app connections, biometric data, children's data.

Strong backbone — drafting work is mostly filling in specifics, not restructuring.
